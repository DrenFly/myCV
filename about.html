Web Development Profile

I prefer to use a Debian Linux platform. I use VS Code and Sublime-Text mainly but also use Atom when dealing with more than one project at a time. Normally I will use terminal based tools via NPM and Node.js. For larger projects I use Gulp.js. I prefer to keep reliance on JavaScript to a minimum and to use as many SASS/CSS solutions as I can for security and reliability reasons.

I also like to avoid frameworks such as bootstrap and jQuery to keep the sites I build as light as possible and to avoid monotony of appearance. Also for reasons of learning to code specific features myself.

Cyber Security & Data Privacy

I am a firm believer in taking some basic security precautions for all users to avoid difficulty no matter the level of threat. However depending on the threat model you fit with there are other important tools available that may be ncessary. I have put together some of my recommendations on the tools page of this site so if you feel you are interested in taking more active measures please have a look here

For a very thorough discussion on the subject and far more information I would recommend visiting the Restore Privacy website. They have many informative reviews and useful advice for your considerations.

Note:

If you don't already know the links I encourage you to explore their value. They are some of the handy online resources that I tend to use and are only suggestions. Everyone has their own set of favourites but searching the web with questions should lead you to the answer, or 10 answers and sometimes a lot more. This is the main way I learn.

CSS Tricks is great for new styling solutions. Rapid Table, MDN and W3C are all great technical reference sites. Google Fonts and Font Awesome are excellent for your site typography. Codepen and Github contain a lot of example code to explore. Stack Overflow and ES Lint will help you solve issues. Node and Gulp will familiarise you with great automation tools. Unsplash is a great source of images. Clippy helps you quickly get the code you need for creating custom shapes. When testing your site Can I Use may clarify why something isn't working by checking which browsers may need a polyfil to work.

I would also recommend courses from Jonas Schmedtmann & Brad Traversy on Udemy and Gary Simon at Design Course.
